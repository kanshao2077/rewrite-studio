# ✍️ Rewrite Studio

把改写语料收进一个工作台。

Rewrite Studio 是一个给改写、仿写、整理语料用的小工具。你可以把参考案例、待改写素材、系统提示词放在一起管理，然后一键拼成完整语料，交给 AI 继续处理。

## 🤯 为什么做它

改写最麻烦的不是让 AI 写，而是每次都要重新拼上下文。

- ❌ 参考案例散在不同文档里，用的时候到处复制。
- ❌ 新素材、参考范文、系统提示词混在一起，越改越乱。
- ❌ 想稳定复用某种风格，却每次都要重新解释一遍。
- ❌ 改写项目一多，就分不清哪个案例属于哪个任务。

Rewrite Studio 的思路很直接：把“参考案例 + 新素材 + 系统提示词”分开管理，最后一键组装。

## 🎯 它适合什么场景

- 做爆款文案、口播稿、短视频脚本的风格迁移。
- 整理一套固定写作风格，反复喂给 AI。
- 给不同客户、栏目、账号保存不同改写项目。
- 对比原文和成品，沉淀可复用的参考案例。
- 把改写语料打包复制到其他 AI 平台。

## 💡 核心功能

- ✅ 多项目管理：不同改写任务分开保存。
- ✅ 参考案例库：每个项目可保存多组原文和成品。
- ✅ 待改写素材：单独维护这次要处理的新内容。
- ✅ 系统提示词：每个项目可保存自己的改写指令。
- ✅ 一键预览复制：自动拼成完整 Prompt 指令包。
- ✅ 本地自动保存：数据保存在当前浏览器里。
- ✅ 备份导出：可以导出项目数据，避免只存在浏览器里。

## ⚙️ 使用方式

1. 新建一个改写项目。
2. 把参考原文和参考成品存成案例。
3. 填入本次要改写的新素材。
4. 调整系统提示词。
5. 点击一键预览复制，把完整语料丢给 AI。

## 💾 数据说明

这个工具没有后端数据库，数据存在你自己的浏览器本地。

换电脑、换浏览器、清理浏览器数据，都可能导致内容丢失。重要项目请及时导出备份，或者复制到你自己的文档里保存。

## 🛠️ 本地运行

需要先安装 Node.js，建议 20 或以上版本。

```bash
npm install --no-package-lock
npm run dev
```

打开终端里显示的网址，就能在本地预览。

## 📦 打包

```bash
npm run build
```

打包结果会生成在 `dist` 目录。
